#### 盒子模型

盒子模型已经称为如今网页设计的趋势，其思想就是把内容放在盒子中，可对盒子的边框等设置样式。

##### 盒子模型的组成

盒子模型有内边距(padding)、外边距(margin)、盒子内容(content)、边框(border)

![2021-03-04_17-39-44.png](http://img.sinkcode.cn/2021/03/04/0d8fa184cc09d.png)

需要注意的点：

1. 可用`border-collapse: collapse;`来实现表格边框重叠的问题

2. 如果只设置盒子的padding，盒子大小可能会被内容“撑”大，但是并不是所有的“撑”大问题都不好，比如：

   ![2021-03-05_10-57-40.png](http://img.sinkcode.cn/2021/03/05/4ddd5b8f8bb15.png)

   在这里有些文字多，有些文字少，如果都设置统一的盒子大小并设置内边距看起来不是很协调，如果不设置盒子大小，直接设置内边距，让它撑开，就会协调很多：

   ![2021-03-05_11-05-37.png](http://img.sinkcode.cn/2021/03/05/a3aa44a6fccc0.png)

3. padding参数多少问题：

   | padding: 5px;                | 上下左右各5像素                       |
   | ---------------------------- | ------------------------------------- |
   | padding: 5px  10px;          | 上下5像素，左右10像素                 |
   | padding: 5px 10px 20px;      | 上5像素，左右10像素，下20像素         |
   | padding: 5px 10px 15px 20px; | 上5像素，左10像素，右15像素，下20像素 |

   即全写的话按照顺时针的顺序。

4. 外边距典型应用——让一个盒子水平居中：`width: 400px; margin: 0 auto;`。如果让行内元素或者行内块元素水平居中，使用`text-align: center;`。

##### 经典问题之边距坍塌（塌陷）

我们先来看一段代码：

```html
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        .father {
            width: 400px;
            height: 400px;
            background-color: rgb(141, 141, 211);
            margin-top: 20px;
        }
        
        .son {
            width: 200px;
            height: 200px;
            background-color: rgb(102, 156, 163);
        }
    </style>
</head>
<body>
    <div class="father">
        <div class="son"></div>
    </div>
</body>
</html>
```

有两个盒子，一个父盒子，一个子盒子，父盒子设置了上外边距20px，此时效果如下：

![2021-03-08_17-01-35.png](http://img.sinkcode.cn/2021/03/08/d825f651e3a20.png)

如果我们给子盒子也设置一个外边距`margin-top: 100px;`，此时效果如下：

![2021-03-08_17-05-18.png](http://img.sinkcode.cn/2021/03/08/0368a56c297dd.png)

可以发现父盒子也被“顶”了下来，这种现象就叫做**边距塌陷**

###### 解决方案

1. 为父盒子定义边框
2. 为父盒子定义内边距
3. 为父盒子添加`overflow: hidden;`

##### 清除默认内外边距

谷歌浏览器会给一些元素设置默认的内外边距，比如`<body>`标签，此时需要用如下清除：

```css
* {
    margin: 0;
    padding: 0;
}
```
